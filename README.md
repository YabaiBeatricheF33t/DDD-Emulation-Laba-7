# Лабораторная работа 7

## Архитектура, слои и DDD-lite

### Описание
Реализация системы оплаты заказа с разделением по слоям и доменной моделью.

### Что реализовано
1. **Доменный слой** - Order, OrderLine, Money, OrderStatus с бизнес-правилами
2. **Слой приложения** - Use Case оплаты заказа
3. **Инфраструктурный слой** - InMemory репозиторий и Fake платежный шлюз
4. **Тесты** - 6 тестов, покрывающих все бизнес-правила

### Бизнес-правила
- Нельзя оплатить пустой заказ
- Нельзя оплатить заказ повторно
- После оплаты нельзя менять строки заказа
- Итоговая сумма равна сумме строк
